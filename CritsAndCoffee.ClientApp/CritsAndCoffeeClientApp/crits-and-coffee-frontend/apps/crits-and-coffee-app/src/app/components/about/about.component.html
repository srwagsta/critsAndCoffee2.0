Jenkins to stay Agile Never. Stop. Progress.

The world is constantly moving, spinning, creating, and rethinking, to keep pace Jenkins just builds. Linked with Git Jenkins makes CI/CD easy. Multi-branch development makes it easy to keep building while staying confident that the production site remains stable. Jenkins rebuilds on each push with the newest Docker images, keeping the project up to date, the tests are run, the containers are pushed to the repository, and if the master is updated Jenkins will take care of redeploying the site out to production. Setup programmatically Jenkins is updated with the project, leaving the development team to do one thing, develop.

Docker + AWS Cloud Native, Ready To Grow

Docker gives an application to scale and move as needed. Every part of this project lives inside containers. During development this give the project the flexibility to move machines, have a full DB that can be replicated and destroyed over and over, most importantly the environment is always the same. More than that, containers can fail, be thrown away, upgraded, segmented, all without worry of data corruption. As the need arises spin up more, and when all is calm take them down to keep costs low. All the containers are deployed and manged using docker-compose with a Kubernetes layer orchestrating everything.

Cloud native from the ground up. AWS provides the infrastructure to manage everything from domain names, containers, networking, and DB availability and replication. Easy to manage and control, but by no mean the only option. The lightweight and portable nature of this project gives it the ability to move and deploy on any cloud platform.

Caddy + Let's Encrypt + CertBot HTTPS That You Can Trust. In Seconds.

when containers are spinning up and down at any moment, ip address are never guaranteed and HTTP proxy is a must. With all the options it becomes hard to choose, but at the end of the day it just needs to do the job. Caddy comes out of the box with HTTPS provided at no cost thanks to Let's Encrypt, and it will always be valid thanks to CertBot! All in containers, it can easily be replaced but with an easy config to keep, and the ability to run even in development there is no need to look further.

PostgreSQL Open Source, Scalable, Relational

When it comes to the data layer there are a ton of options. NoSql didn't fit the job here. Users were a must, and there are plan to build more relationships later in the project. So relational was a must. PostgreSQL is open source, growing rapidly, and has the perfect GIS support. Simple, supported, solutions.

.NET Core Multi-platform, Fast, Familar

On the API side there are options, probably to many, but when it comes to a well supported platform that has a massive knowledge base .Net was the clear choice. This is project is a sandbox for real world skills, .Net has the largest user base and is the clear choice for enterprise, so keeping those skills sharp is exactly what I need. Now muilt-platform thanks to Core, and Angular ready thanks to the added support for SPA (Single Page Applications). All of the functionality I need with the speed of a compiled language, and the great features of the C# language!

Flask Lightweight, With The Data Stomping Power Of Python

Flask is a micro-framework built on python, with low overhead, and extensive libraries. This small python web framework allows me to use the latest Python data libraries while still inside a web capable framework. Of course this means I can leverage Anaconda Python distribution when I need to, small services ready for big data.
